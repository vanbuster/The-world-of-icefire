# 《维斯特洛互动世界地图》项目约束文件

> 状态：项目总控约束
> 创建日期：2026-06-08
> 维护规则：后续每次出现重大要求、范围变化、技术路线变化、版权边界变化或阶段确认结果，都必须同步更新本文件。

## 1. 项目定位

- 项目名称：《维斯特洛互动世界地图》
- 项目类型：网页端互动地图 / 世界观图鉴 / 个人二创作品
- 发布形式：公开网站
- 发布目的：个人作品展示与社区分享
- 商业属性：非商业运营
- 版权原则：避免直接搬运官方地图、官方剧照、大段原文、未经授权商业素材；优先使用原创风格资产、明确授权素材或可合理使用的参考资料。

## 2. 内容范围

- 第一阶段仅覆盖维斯特洛大陆。
- 后续可扩展：
  - 厄索斯
  - 索斯罗斯
  - 长城以北
  - 其他世界区域

## 3. MVP 范围

- MVP 只做“临冬城”一个可点击详情。
- 临冬城详情应作为完整样板，包含基础介绍、关联人物、关联事件、图片素材、时间线、局部地图。
- 理想状态下，每个故事、每个阶段都能关联配图；MVP 可先在临冬城样板中验证此能力。
- 其他地点先作为地图节点展示，不要求完整详情页。
- 地图需要展示的内容类型：
  - 关键城市
  - 城堡
  - 地区
  - 海域
  - 道路
  - 河流
  - 山脉
  - 森林
  - 长城

## 4. 内容与素材管理

- 允许使用资料、插画、手绘地图素材作为参考或素材来源。
- 地图与视觉资产来源候选：
  - 原创绘制
  - AI 辅助生成
  - 参考公开资料后重绘
  - 开源 / 授权手绘地图
  - 基于剧集或插图参考进行 AI 二次创作，但必须注意版权风险与公开发布边界
- 后台 CMS 必须保留素材元数据：
  - 素材来源
  - 授权状态
  - 作者
  - 链接
  - 备注
  - 使用位置
  - sourceType 或等价来源分类
- 第一版先做基础来源记录，后续可增强为更严格的授权与版权合规字段。
- 素材策略：尽量使用原创风格资产，不复制官方视觉资产。

## 5. 系统形态

- 必须包含后台 CMS。
- CMS 是长期可维护内容管理系统，不是一次性静态数据表。
- CMS 第一版目标接近正式编辑后台，支持列表筛选、草稿状态、发布状态、预览、关系字段、素材库管理。
- 后台第一版只需要单管理员账号登录。
- 后续需要可持续添加：
  - 地点
  - 人物
  - 事件
  - 家族
  - 图片
  - 地图素材
  - 故事线
- MVP 只开放地点与素材编辑；人物、事件、家族、故事线先预留内容模型。

## 6. 剧情与内容规则

- 小说与剧集混合。
- 原著已有内容优先忠于原著。
- 原著没有的情节可以使用剧集事件。
- 所有内容需要标记 `sourceType`：
  - `novel`
  - `show`
  - `show-canon`
  - `mixed`
  - `original-note`
- 示例：“私生子之战”应标记为 `show` 或 `show-canon`。
- 第一版不做小说线 / 剧集线切换 UI，仅在内容字段中标记 `sourceType`。

## 7. 端适配

- 当前只做桌面端。
- 暂不做移动端适配。
- 优先保证大屏沉浸式地图体验。

## 8. 视觉目标

项目视觉风格应参考：

- 高保真
- 中世纪幻想地图
- 牛皮纸 / 羊皮纸 / 古地图质感，但不局限于平面线稿地图
- 类 3D 微缩景观地图
- 立体地形、微缩城堡、山脉、森林、河流、道路与海岸
- 可作为后续迭代再创作的幻想地理沙盘
- 暗金色 UI
- 博物馆级世界观图鉴
- 家族纹章
- 地点节点
- 云雾天气层
- 北境雪雾
- 战争烟尘
- 右侧详情面板
- 时间线
- 人物卡片
- 局部地图
- 故事主线
- 强动效幻想沙盘；如实现难度或性能成本过高，可降级到标准动效。
- 搜索体验后续应向“百科全书式统一检索”发展，覆盖地点、人物、事件、家族。

## 9. 默认技术方向

### 前台

- Next.js App Router
- React
- TypeScript
- Tailwind CSS
- Framer Motion
- Zustand

### 后台

- Payload CMS
- PostgreSQL
- 媒体资源存储先本地
- 后续可扩展到 S3 / Cloudflare R2

### 地图

- MVP：可从高精地图底图 + HTML/SVG 节点层 + CSS/Canvas 天气层起步，但架构必须预留类 3D 微缩景观渲染能力。
- 后续增强：PixiJS / WebGL / Three.js，支持地形高度、立体地点模型、动态天气、镜头运动与分层渲染。
- 视觉迭代方向：从“古地图平面浏览”逐步演进为“可探索的维斯特洛微缩沙盘”。
- 地图节点坐标是否在后台可视化编辑暂未确认；PRD / 技术架构阶段需要解释成本与收益后再定。

### 部署

- 本地开发优先
- 面向公开网站发布
- 后续可部署到 Vercel / Railway / Render 等平台

## 10. 阶段门禁

不得直接开始写代码。必须按以下阶段逐级推进：

0. 需求澄清与默认假设
1. PRD 产品需求文档
2. 信息架构 IA 与内容模型
3. 技术架构文档
4. 视觉与素材指南
5. 详细技术文档
6. 开发任务拆解
7. Codex 实现提示词包
8. 进入代码实现

每个阶段完成后，必须先输出阶段文档，并等待用户确认。

用户说“继续”或“使用默认方案”后，才可以进入下一阶段。

## 11. 当前默认假设

- 项目以中文内容为主，必要时预留英文名、别名、原文名字段。
- 第一版地图底图应为原创或授权可用的“风格化维斯特洛大陆地图”，不直接使用 HBO、书籍官方地图或商业游戏素材。
- 前端体验不应被限定为羊皮纸平面地图；PRD 与技术架构阶段需要明确类 3D 微缩景观地图的迭代路线。
- MVP 可以先实现 2.5D / 伪 3D 表现，例如等距视角、地形浮雕、分层景深、动态雾层、微缩建筑图标；后续再演进到 WebGL / Three.js 真 3D 或 PixiJS 高性能 2.5D。
- “临冬城”详情作为首个内容样板，承担地点详情页、人物关联、事件关联、素材引用和剧情来源标注的验证作用。
- 后台优先保证内容结构正确与可维护，前台优先保证强沉浸式地图体验。
- 首阶段不处理用户系统、登录注册、评论、多人协作编辑、商业化和移动端。
- “用户系统”不包含后台单管理员登录；后台单管理员登录属于 MVP 基础能力。

## 13. 需求变更记录

### 2026-06-21：V1.8 引擎资产管线与高保真 2.5D 验证要求

- 用户确认：当前可行路线应以 C 方案为主，即“Unity / Unreal / Blender / Spline 等引擎或 3D 工具负责资产编辑，网页负责最终运行、CMS、详情面板、搜索和部署”。
- 用户理由：
  - Unity / Unreal 质感、交互性、可玩性上限更高。
  - 引擎场景可局部修改地形、城市、材质、灯光，不必整张地图重绘。
  - 模块化资产比单张生成图更适合长期维护。
  - 但用户没有 3D 建模 / 引擎经验，因此不能直接把整个项目押到完整游戏工程。
- V1.8 新循环正式命名为：《维斯特洛互动世界地图 V1.8 引擎资产管线与高保真 2.5D 验证循环》。
- V1.8 状态目录为：`docs/westeros-interactive-map/loop-v1-8-engine-asset-pipeline/`。
- V1.8 顶层提示为：`docs/westeros-interactive-map/V1_8_ENGINE_ASSET_PIPELINE_LOOP_PROMPT.md`。
- `docs/westeros-interactive-map/ACTIVE_LOOP.md` 已更新为指向 V1.8 loop。
- V1.8 推荐执行强度为“标准验证”：
  - 主线：引擎 / 3D 工具生产资产，Web 负责运行与 CMS。
  - 附带验证：评估 Unity WebGL / 轻 3D 是否值得后续继续投入。
  - 样例区域：北境 / 临冬城沙盘 tile。
- V1.8 不直接重做完整维斯特洛大陆，不直接迁移为完整 Unity / Unreal 项目。
- V1.8 必须验证：
  - 引擎化 / 3D 资产质感是否明显强于当前 V4 单张底图。
  - 地形、城市、森林、道路、材质、天气能否模块化维护。
  - 资产能否导出为网页可运行格式，例如 glTF、PNG tile、sprite atlas、heightmap 或等价格式。
  - 网页端能否加载样例资产并与现有地图 UI、地点锚点、详情面板兼容。
  - CMS 后续如何管理 TerrainTile、LandmarkPrefab、MaterialSet、RouteLayer、WaterLayer、WeatherPreset、MapAnchor 等对象。
  - 如果用户不会建模，是否仍能通过 AI、程序化生成、模板、现成资产和轻量 3D 工具完成迭代。
- 若验证显示成本过高，应降级回“高完成度平面 / 2.5D 网页作品”路线；降级不视为失败。

### 2026-06-20：V1.6 原著地理约束与外挂知识库要求

- 用户反馈：当前沙盘底图中有大量意义不明的城市和建筑，破坏地点可信度；鹰巢城应位于险要峡谷 / 高山，君临应贴近河流与海湾，高庭应位于遥远的河湾地而不是贴近王都。
- 新增外挂知识库文档：`docs/westeros-interactive-map/WESTEROS_GEOGRAPHY_KNOWLEDGE_BASE.md`。
- 后续生成或替换全局沙盘底图前，必须先对照该知识库中的地理锚点、禁令、地点审查表和生成 prompt。
- 底图生成策略调整：
  - 背景图主要表达大陆轮廓、山脉、河流、海岸、森林、雪线、道路方向和少量关键地貌。
  - 地点含义由前端节点、tooltip、详情面板和 CMS 数据承担。
  - 禁止在底图里随机生成无名城市、无意义城堡群或无法解释的地标。
  - 新底图必须明确鹰巢城 / 君临 / 高庭 / 奔流城 / 赫伦堡 / 派克 / 阳戟城等关键地点的地貌关系。
- V1.6 loop 增加地理知识库与原著约束版底图验收项；V2 高精底图不再视为最终完成，需升级为 V3 原著地理约束版。
