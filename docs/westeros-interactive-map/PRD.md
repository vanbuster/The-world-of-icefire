## 1. 产品定位

### 1.1 产品一句话描述

《维斯特洛互动世界地图》是一个以地图为核心入口的《冰与火之歌 / 权力的游戏》世界观浏览器，让用户通过地点、事件、人物、家族和时间线理解维斯特洛大陆的故事脉络。

### 1.2 项目性质

- 个人二创作品
- 面向社区公开发布
- 非商业运营
- 第一版发布形态为公开网站
- 第一阶段聚焦维斯特洛大陆
- 需要保留素材来源信息，避免直接搬运官方长文本或未经授权的官方视觉素材

### 1.3 目标用户

- 《冰与火之歌》原著读者
- 《权力的游戏》剧集观众
- 世界观设定爱好者
- 地图、家族、战争线索研究者
- 想快速回顾剧情与地点关系的用户
- 后续维护内容的项目作者

### 1.4 使用场景

- 通过地图浏览维斯特洛关键地点
- 点击临冬城查看该地点相关人物、事件和故事阶段
- 沿时间线回顾临冬城的重要剧情
- 通过素材图集获得更具沉浸感的视觉理解
- 在后台维护地点、素材、来源记录和发布状态

### 1.5 核心价值

这个产品不是普通剧情百科，而是一个 **地图驱动的世界观浏览器**。

核心价值在于：

- 用地图建立空间感
- 用地点串联人物、家族和事件
- 用时间线展示故事阶段变化
- 用视觉氛围强化历史厚重感和幻想感
- 用 CMS 支撑长期维护和内容扩展

### 1.6 非目标范围

MVP 不做：

- 移动端适配
- 商业运营
- 前台用户注册、评论、收藏
- 多管理员权限系统
- 小说线 / 剧集线前台切换 UI
- 完整时间线筛选
- 完整百科搜索系统
- 真 3D 地图引擎
- 拖拽式地图编辑器
- 厄索斯、索斯罗斯、长城以北等区域

## 2. 核心产品决策

### 2.1 内容范围

第一阶段只做维斯特洛大陆，不做厄索斯。

MVP 地图上展示一批关键地点节点，其中只有临冬城拥有完整详情内容。其他地点只作为地图节点展示，支持 hover 状态、名称展示和基础类型展示。

后续可扩展：

- 更多地点详情
- 人物
- 家族
- 事件
- 战争
- 故事线
- 时间线
- 多地图层

### 2.2 MVP 范围

MVP 聚焦完成一个完整闭环：

```text
进入公开网站
→ 浏览维斯特洛地图
→ hover 地点节点
→ 点击临冬城
→ 查看完整详情
→ 通过后台维护地点与素材
```

### 2.3 发布形态

第一版目标是公开网站，可供社区访问和分享。

### 2.4 剧情来源规则

- 原著已有内容优先忠于原著
- 原著没有或尚未发生的内容，可用剧集事件补充
- 二创说明必须明确标记为原创注释
- 所有内容必须具有 `sourceType`

推荐来源类型：

- `novel`
- `show`
- `show-canon`
- `mixed`
- `original-note`

### 2.5 小说线 / 剧集线 / 混合线区分方式

MVP 不做前台显式切换 UI。

第一版只在事件、地点详情或内容卡片中展示小型来源标签。后续版本再提供“小说线 / 剧集线 / 混合线”切换。

### 2.6 CMS 后台长期维护策略

后台不是临时 JSON 编辑器，而是接近正式编辑后台：

- 单管理员登录
- 内容列表
- 搜索 / 筛选
- 草稿状态
- 发布状态
- 预览能力
- 关系字段
- 素材库管理
- 地点内容管理
- 素材上传与来源记录

MVP 重点完成 Locations 与 Assets。其他内容类型先预留。

### 2.7 桌面端优先策略

当前只做桌面端。优先保证大屏地图浏览、右侧详情面板和沉浸式视觉体验。

### 2.8 强动效幻想沙盘策略

视觉目标为强动效幻想沙盘，但 MVP 不强制真 3D。第一版推荐 2.5D 高保真地图体验。

标准目标：

- 云雾流动
- 北境雪雾
- 战争烟尘
- 节点微动效
- 地图景深
- 鼠标视差
- 详情面板转场

性能不足时允许降级：

- 低动效模式
- 关闭高成本粒子
- 减少 Canvas 图层
- 使用 CSS 动画替代复杂渲染
- 保证普通桌面浏览器可运行

### 2.9 素材来源记录策略

素材第一版做基础来源记录，不做过重版权法务系统。

必须记录：

- 素材名称
- 素材类型
- 来源类型
- 原始来源链接
- 作者 / 来源方
- 是否 AI 生成
- 是否二次创作
- 备注

不建议直接使用未经授权的官方地图原图作为公开网站核心素材。建议采用“参考风格 + 重绘 + AI 辅助生成 + 来源记录”的方式。

### 2.10 后续百科搜索策略

搜索是后续的“维斯特洛百科全书入口”。

未来搜索对象包括：

- 地点
- 人物
- 事件
- 家族
- 战争
- 故事线
- 素材

MVP 可只保留搜索入口或轻量地点搜索。

## 3. 用户故事

1. 作为读者，我想点击临冬城，查看它在故事中的所有关键事件。
2. 作为世界观研究者，我想按家族势力理解地点与人物关系。
3. 作为剧情回顾者，我想沿时间线查看临冬城相关事件发生顺序。
4. 作为视觉体验用户，我想在地图中感受到天气、战争、历史和幻想氛围。
5. 作为内容维护者，我想通过后台添加地点、素材和来源信息。
6. 作为未来内容维护者，我希望后续可以扩展人物、事件、家族和故事线。
7. 作为百科检索用户，我希望未来可以统一搜索地点、人物、事件、家族和故事线。

## 4. 功能模块

| 模块 | 模块目标 | MVP 是否需要 | 核心功能 | 后续扩展方向 |
|---|---|---|---|---|

| 地点节点模块 | 在地图上呈现关键地点 | 必做 | 至少 10 个节点、hover、类型展示 | 节点分级、缩放可见范围、动态状态 |
| 城市详情面板 | 展示地点详情内容 | 必做 | 右侧面板、简介、素材、关联信息 | 多地点详情、深度百科页 |
| 临冬城详情模块 | 完成 MVP 内容闭环 | 必做 | 临冬城完整内容样板 | 扩展到其他地点 |
| 临冬城局部地图模块 | 呈现临冬城内部或周边空间 | 必做 | 局部图、关键区域标注 | 可交互局部地图 |
| 时间线模块 | 展示事件顺序 | 必做，限临冬城 | 临冬城事件时间线 | 按卷、季、战争、人物筛选 |
| 事件详情模块 | 展示事件内容与来源 | 必做，限临冬城事件 | 事件卡片、来源标签、配图 | 独立事件页、关系网络 |
| 人物卡片模块 | 展示地点关联人物 | 必做，限临冬城 | 人名、身份、家族、关联事件 | 人物详情页、生命线 |
| 家族势力模块 | 表达地点与家族关系 | 必做，基础展示 | 史塔克等相关家族信息 | 家族势力变化、纹章系统 |
| 天气与云雾模块 | 增强沉浸感 | 必做，基础版 | 云雾、雪雾、烟尘、氛围动画 | 动态天气、区域状态 |
| 搜索模块 | 提供未来百科入口 | MVP 可轻量 / 预留 | 搜索入口、可搜索地点 | 统一百科搜索、全文检索 |
| 图例模块 | 帮助理解节点类型 | 必做 | 地点类型、节点等级说明 | 可筛选图层 |
| 内容数据模块 | 组织前后台内容关系 | 必做 | 地点、素材、事件来源标记 | 完整知识图谱 |
| 后台 CMS 模块 | 支撑长期维护 | 必做 | 登录、列表、编辑、预览、发布状态 | 多角色、审核流程 |
| 素材管理模块 | 管理图片和来源 | 必做 | 上传、来源记录、关联地点 | 授权状态、素材审计 |
| 来源标记模块 | 区分小说、剧集、混合和二创 | 必做 | `sourceType` 标签 | 前台线路切换 |
| 后续百科搜索模块 | 发展为世界观检索入口 | 预留 | 规划入口 | 类型筛选、全文搜索、结果跳地图 |

## 5. MVP 范围

第一版 MVP 建议实现：

- 公开网站前台首页
- 维斯特洛大陆全局地图
- 10 个地点节点
- 临冬城一个完整可点击详情
- 基础缩放和平移
- 基础云雾 / 雪雾 / 氛围动画
- 临冬城事件时间线
- 临冬城局部地图
- 临冬城关键事件卡片
- 临冬城相关人物卡片
- 临冬城相关家族展示
- 每个关键故事阶段尽量配图
- 事件来源标记
- 素材来源记录
- 后台单管理员登录
- 后台可维护 Locations
- 后台可维护 Assets
- 其他内容类型预留模型
- 桌面端优先布局
- 不做移动端适配
- 不做小说线 / 剧集线切换 UI
- 不做复杂时间线筛选
- 不做完整地图节点拖拽编辑器

## 6. 临冬城 MVP 内容

### 6.1 地点内容

临冬城第一版需要包含：

- 地点简介：北境核心城堡，史塔克家族祖传居城
- 地理位置：维斯特洛北境腹地，国王大道北段重要节点
- 政治归属：北境，史塔克家族
- 相关家族：史塔克、拜拉席恩、葛雷乔伊、波顿等
- 关键人物：艾德、凯特琳、罗柏、珊莎、艾莉亚、布兰、琼恩、席恩、拉姆斯等
- 关键事件：至少 7 个
- 局部地图：临冬城及周边区域示意
- 图片素材：城堡、雪境、庭院、战争后场景等
- 事件时间线：按故事顺序展示
- 事件来源标记：每个事件必须有 `sourceType`
- 相关故事线：北境线、史塔克家族线、五王之战线、剧集北境收复线
- 视觉氛围建议：寒冷、雪雾、旧石城、狼纹章、战争痕迹、北境肃穆感

## 8. 用户流程

### 8.1 前台浏览流程

```text
进入公开网站首页
→ 查看维斯特洛全局地图
→ 观察云雾、天气和地图节点
→ 悬浮地点节点
→ 查看地点名称和基础信息
→ 点击临冬城
→ 打开右侧详情面板
→ 查看临冬城介绍
→ 查看局部地图
→ 查看历史时间线
→ 点击关键事件
→ 查看事件配图、人物和家族
→ 查看来源标记
→ 返回全局地图
```

### 8.2 后台维护流程

```text
管理员登录
→ 进入 CMS
→ 新建或编辑地点
→ 上传素材
→ 填写素材来源
→ 关联地点与素材
→ 设置草稿 / 发布状态
→ 预览前台效果
→ 发布内容
```

## 9. 后台 CMS 需求

### 9.1 内容类型

| 内容类型 | 用途 | MVP 状态 |
|---|---|---|
| Locations | 管理地点、坐标、详情、关联素材 | 必做 |
| Regions | 管理北境、河间地、谷地等区域 | 预留 |
| Houses | 管理家族、纹章、势力关系 | 预留 |
| Characters | 管理人物、身份、家族、事件关系 | 预留 |
| Events | 管理事件、来源、时间线、关联人物 | 预留 |
| Assets | 管理图片、地图素材、来源记录 | 必做 |
| MapLayers | 管理道路、河流、山脉、森林等地图层 | 预留 |
| WeatherLayers | 管理云雾、雪雾、烟尘等氛围层 | 预留 |

## 10. 数据字段需求概览

以下为 PRD 层面的字段需求描述，不是数据库 Schema。

### 10.1 Locations

Locations 用于管理地图上的地点节点和地点详情。

关键字段包括：

- 名称
- 英文名
- 地点类型
- 所属区域
- 所属家族
- 简介
- 地图坐标 `xPercent`
- 地图坐标 `yPercent`
- 图标类型
- 节点等级
- 是否 MVP 可点击
- 详情内容
- 关联素材
- 发布状态

关于坐标编辑：

MVP 不做复杂拖拽式地图编辑器，但建议保留 `xPercent`、`yPercent`、`iconType`、`nodeLevel`、`visibleZoomRange` 等字段。前台根据百分比坐标渲染节点，后台先手动填写，后续再升级为可视化拖拽编辑。

### 10.2 Assets

Assets 用于管理图片、插画、局部地图、AI 生成素材和来源记录。

关键字段包括：

- 素材名称
- 素材类型
- 文件
- 来源类型
- 来源链接
- 作者 / 来源方
- 是否 AI 生成
- 是否二次创作
- 关联地点
- 关联事件
- 备注

### 10.3 Events

Events 用于管理故事事件，MVP 可先作为预留模型或临冬城内容的一部分。

关键字段包括：

- 事件名称
- 简介
- 发生地点
- 相关人物
- 相关家族
- `sourceType`
- 时间线排序
- 关联素材
- 是否剧集线事件
- 是否原著事件
- 是否二创注释
